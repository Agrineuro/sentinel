# FOUNDATION

> *Software should adapt to people—not the other way around.*

---

# Why Sentinel Exists

Modern self-hosted environments are more powerful than ever.

They are also more fragmented than ever.

A single homelab may contain virtualization platforms, storage servers, containers, smart home systems, AI models, monitoring tools, media servers, automation engines, and dozens of web interfaces. Each has its own dashboard, authentication, terminology, and workflow.

The more capable a homelab becomes, the more time its owner spends managing software instead of enjoying it.

Sentinel exists to change that.

Sentinel is an open-source operating platform that brings infrastructure, automation, artificial intelligence, and personal workflows together into one cohesive experience.

Rather than replacing existing software, Sentinel connects to it.

Rather than creating another dashboard, Sentinel becomes the environment in which those dashboards work together.

# What Sentinel Is

Sentinel is an operating platform for self-hosted infrastructure.

It combines intelligent automation, infrastructure management, monitoring, artificial intelligence, and personalization into one extensible platform.

Sentinel connects existing technologies instead of replacing them.

Examples include:

- Virtualization platforms
- Storage appliances
- Linux and Windows systems
- Smart home platforms
- Containers
- Artificial intelligence models
- Cameras
- Monitoring systems
- Networking equipment
- Cloud services
- Future technologies not yet imagined

Sentinel should feel less like another application and more like the operating environment that sits above everything else.

---

# What Sentinel Is Not

Sentinel is not another chatbot.

Sentinel is not another monitoring dashboard.

Sentinel is not another SSH client.

Sentinel is not another home automation platform.

Sentinel is not another virtualization manager.

Sentinel is not another container interface.

Each of those products already exists and does an excellent job within its own domain.

Sentinel's purpose is to connect them into one coherent experience.

# Core Principles

## Local First

The user's data belongs to the user.

Every feature should function without requiring cloud services whenever technically possible.

Cloud integrations should always remain optional.

---

## Privacy First

Sentinel should collect only the information necessary to provide its functionality.

The user owns their data.

The user controls their data.

The user decides where their data resides.

---

## Human First

People should not need to memorize commands.

Technology should become more approachable without limiting advanced users.

Simple tasks should remain simple.

Complex tasks should remain possible.

---

## Transparent Intelligence

Artificial intelligence should never become a black box.

Whenever Sentinel performs an action, it should explain what happened, why it happened, and how it reached that decision.

Users should always remain in control.

---

## Progressive Complexity

A beginner should be able to install Sentinel and begin using it within minutes.

An experienced administrator should be able to customize nearly every aspect of the platform.

Sentinel should scale alongside its users rather than forcing them into different products.

---

## Plugin First

Every integration should exist as a connector or plugin whenever practical.

Nothing should be unnecessarily hardcoded.

This allows the platform to evolve alongside the community.

---

## API First

Every capability should be accessible through a documented API.

The web interface, desktop clients, mobile applications, and future interfaces should all consume the same underlying platform.

---

## Secure by Default

Security is a feature.

Encrypted credentials.

Role-based access.

Least privilege.

Audit logging.

Safe defaults.

These are not optional additions—they are foundational requirements.

---

## Community Driven

The best ideas rarely come from one person.

Sentinel should remain approachable for first-time contributors while providing the flexibility required by experienced developers.

Community contributions should shape the future of the platform.

# Looking Forward

Technology changes constantly.

Sentinel is not being built around today's software.

It is being built around timeless principles.

New operating systems will emerge.

New artificial intelligence models will appear.

New protocols will replace old ones.

The platform should evolve alongside them.

Sentinel is intended to become a foundation upon which the community can continue building for many years.

The goal is not to create another application.
